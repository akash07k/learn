# Subject Template: An Accessible, Shell-Only Guide Corpus

This is the skeleton for a new `learn` subject. Copy `subjects/_template/` to
`subjects/<your-subject>/`, rename it, and replace this content with your own. Every guide follows
the repository-root `AUTHORING-CONVENTIONS.md`: it is written for a screen-reader reader, taught
with exact commands and file paths, and ends with its sources.

## How to use these guides

- Read the numbered guides in order. They build on each other.
- Each guide is self-contained: it states what you will be able to do, teaches it with exact
  commands, gives a "Verify it worked" check, and ends with its sources.

## Reading order

- [Welcome: what this subject is](00-welcome.md) - the map before the hands-on work.
- [Your first task](01-first-task.md) - the first concrete thing you will do.

## Where else to look

- [GLOSSARY.md](GLOSSARY.md) - the canonical definition of every term used here.
- `CONTEXT.md` at this subject's root - the glossary of roles and vocabulary.
- `AUTHORING-CONVENTIONS.md` at the repository root - the conventions every guide follows. Add an
  `AUTHORING-NOTES.md` at this subject's root for any domain-specific safety, version, or lint
  conventions.
